# Content Strategy

Goal: present `marketdata_cpp` as a clear, functioning system to technical and business audiences.

Principles:
- Show validation status first (run status, deterministic check, key metrics)
- Keep visuals autoplay/passive to reduce friction
- Always expose raw artifact links for auditability
- Explain why each metric/check matters in plain language

Page roles:
- Home: one-screen system summary
- Replay: visual market-state snapshots
- Performance: latency/throughput interpretation
- Correctness: deterministic and invariant validation
- Architecture: pipeline and provenance
- Artifacts: raw downloadable files
